## Screen Contracts

### Screen 1: /login

**Layout:** Full-viewport centered flex. Background `--background`. Card: max-width 380px, `--card` background, `border border-border`, `rounded-2xl`, padding `xl` (32px) horizontal / `2xl` (48px) vertical equivalent.

**Component map (shadcn):**
- Outer container: plain div with Tailwind `min-h-screen flex items-center justify-center bg-background`
- Card: shadcn `Card` + `CardContent` (replaces inline div)
- Logo chip: `<span>` with `bg-primary text-primary-foreground rounded-lg px-4 py-2 font-sans font-extrabold text-base`
- Email field: shadcn `Label` + `Input` (type="email", autoComplete="email")
- Password field: shadcn `Label` + `Input` (type="password", autoComplete="current-password")
- Submit: shadcn `Button` variant="default" (maps to `bg-primary text-primary-foreground`) full width
- Error: shadcn `Alert` or inline div with `bg-destructive/10 border border-destructive/30 text-destructive text-[12px] font-mono`

**States:**
- Default: empty fields, "Entrar" button enabled
- Loading: button shows "Entrando..." text, `disabled` prop, `opacity-70`
- Error: error div visible above button with message, fields remain editable
- No auto-signup link, no "Esqueci minha senha" link (out of scope for this phase)

**Branding:** "DA SILVA" chip always visible at top center. "Painel de Vendas" heading below chip (20px Syne weight 700). Subtext "Entre com suas credenciais" (12px DM Mono muted).

### Screen 2: /dashboard/usuarios

**Layout:** Standard dashboard layout (sidebar + topbar from Phase 1 foundation). Page content area uses `PageHeader` component with title "Usuários" and `actions` slot containing "Convidar usuário" button.

**Component map (shadcn):**
- Page title: `PageHeader` with title="Usuários", subtitle="Gerencie acessos e convites da equipe"
- Filters row: two `Select` dropdowns (Role, Loja) side by side, left-aligned, gap-sm (8px)
- Users table: shadcn `data-table.tsx` — replaces current manual `<table>`
- Loading: `skeleton.tsx` rows (5 rows × full width) while data fetches
- Edit action: row action opens shadcn `Sheet` (side="right", width 400px)
- Invite action: "Convidar usuário" button opens shadcn `Sheet` (side="right", width 400px)
- Disable confirmation: shadcn `Dialog` with destructive confirm
- Copy link: `Button` variant="outline" with `Copy` icon (lucide), switches to `Check` icon for 1500ms after copy success

**Table columns (AUTH-06 + CONTEXT D-08):**

| Column | Source | Font | Notes |
|--------|--------|------|-------|
| Nome | profiles.name | Syne 14px weight 600 | With `Avatar` initials (24px) if no photo |
| N° Vendedor | profiles.numero_vendedor | DM Mono 12px | `—` when null |
| Loja | lojas.nome via loja_id | `LojaBadge` component | Pre-existing component |
| Role | profiles.role | Badge (role color system above) | 4 roles: vendedor, gerente, adm, super_admin |

**Invite Sheet (right panel, width 400px):**

- Expiry note: "Link válido por 7 dias" (12px DM Mono muted, below the link block)
- Close button: Sheet native X icon (top right)

**Edit User Sheet (right panel, width 400px):**
- Title: "Editar usuário" (Syne 20px weight 700)
- Fields: Nome (Input), Role (Select), Loja (Select), Ativo (Switch with label)
- Save button: "Salvar alterações" — `Button` variant="default" (accent), full width
- Cancel: "Cancelar" — `Button` variant="outline", full width above save (or footer row)

- Title: "Desativar usuário"

### Screen 3: /convite/[token]

**Layout:** Full-viewport centered flex (same shell as /login). No sidebar, no topbar. Public route — unauthenticated.

**Component map (shadcn):**
- Outer: `min-h-screen flex items-center justify-center bg-background px-4`
- Card: `Card` + `CardContent`, max-width 380px (matches /login card)
- Logo chip: same "DA SILVA" chip as /login (brand consistency)
- Heading: "Definir senha" (Syne 20px weight 700)
- Subtext: "Bem-vindo(a) ao Da Silva. Escolha uma senha para acessar o painel." (16px Syne muted)
- Password field: `Label` + `Input` type="password", autoComplete="new-password"
- Confirm field: `Label` + `Input` type="password", autoComplete="new-password"
- Submit: `Button` variant="default" full width, label "Definir senha"
- Error (token invalid/expired): see error states below

**States:**

| State | Trigger | UI |
|-------|---------|-----|
| Valid token | token exists, not expired, not used | Show password form |
| Invalid token | token not found in DB | Full-card error: "Link inválido. Solicite um novo convite ao seu gerente." |
| Expired token | expires_at < now() | Full-card error: "Link expirado. Solicite um novo convite — links são válidos por 7 dias." |
| Used token | used = true | Full-card error: "Este link já foi utilizado. Faça login com suas credenciais." + link to /login |
| Mismatched passwords | client-side validation | Inline error below confirm field: "As senhas não conferem." |
| Password too short | client-side | Inline error: "A senha deve ter pelo menos 8 caracteres." |
| Loading submit | form submitted | Button shows "Definindo..." + disabled |
| Success | password set | Full-card success: "Senha definida com sucesso! Redirecionando..." → redirect to /login after 2s |

**Token error card:** Same Card shell as the form, but replaces form with centered content: error icon (`AlertCircle`, lucide, 40px, `text-destructive`), title (20px Syne weight 700), body text (16px Syne muted), optional link button variant="outline".

**No visible fields for role or loja** — these come pre-configured from the token. User sees only password fields. (Source: CONTEXT D-07)

## Interaction Contracts

### Copy-to-clipboard ("Copiar link")

1. User clicks "Copiar link" button (variant="outline", `Copy` lucide icon left)
2. `navigator.clipboard.writeText(link)` called
3. Button icon switches from `Copy` to `Check`, label changes to "Link copiado"
4. After 1500ms: reverts to original state
5. On clipboard API failure (non-HTTPS, denied): fallback to `document.execCommand('copy')` or show tooltip "Copie manualmente: {link}"

### Disable user flow

1. Row action → "Desativar" menu item (DropdownMenu, `text-destructive`)
2. `Dialog` opens with confirmation text and user name
3. "Desativar" button (variant="destructive") → POST `/api/admin/disable-user` with `{ userId }`
4. API calls `admin.updateUserById({ banned: true })`
5. On success: row updates to `ativo: false` (optimistic update), badge changes to "inativo"
6. On error: Dialog stays open, inline error: "Falha ao desativar. Tente novamente."

### Token validation on /convite/[token] load

1. Server component fetches token from `convites` table (server-side, no client fetch)
2. Checks: token exists AND `expires_at > now()` AND `usado = false`
3. If any check fails: renders error card (no redirect — stay on same URL with clear message)
4. If valid: renders password form with token embedded in hidden field or handled via server action

### Role filtering behavior

- Role and Loja filters use `<Select>` components bound to URL searchParams
- On change: `router.push` with updated searchParams (no form submit)
- Table re-renders with filtered data from server (RSC)
- "Limpar filtros" link: `href="/dashboard/usuarios"` (removes all params)

## Accessibility Contracts

- All form fields: `<Label htmlFor>` linked to `<Input id>` (shadcn enforces this via `form.tsx`)
- Error messages: `role="alert"` on error containers so screen readers announce immediately
- Dialog: focus trapped inside when open (Radix Dialog handles this natively)
- Sheet: focus trapped, ESC closes, focus returns to trigger on close (Radix Sheet)
- "Copiar link" button: `aria-label="Copiar link de convite"` to disambiguate from other copy actions
- Disable button in Dialog: `aria-describedby` pointing to the body text
- Skeleton loading: wrapping div with `aria-label="Carregando usuários..."` and `aria-busy="true"`
- Color is never the sole indicator of status: badges use text label AND color tint
- Table: `<thead>` with `scope="col"` on `<th>` elements
