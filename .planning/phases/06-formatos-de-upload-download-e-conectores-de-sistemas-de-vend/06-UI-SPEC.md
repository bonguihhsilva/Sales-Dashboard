## Interaction Contract

### Export Format Selection

1. User sees 3 format tabs: CSV (pre-selected) / XLSX / PDF
2. User clicks a format tab → active tab border changes to neon-border-active, no page reload
3. User clicks "Relatório [FORMAT]" button → loading state (hourglass icon, "Gerando...") → file download triggers → button returns to idle
4. Error: toast.error via `sonner` — "Falha ao gerar relatório" + error detail (existing pattern)

### Upload — Format Detection

1. User selects file → existing parse API call triggered
2. If `detected_system` not null: badge appears ("Formato detectado: CEC") — upload proceeds as today
3. If `detected_system` is null: dropdown appears — user selects system → upload proceeds with selected system passed in request body
4. Fingerprint badge and dropdown appear inside the existing modal body, between the file input and the period/mode selectors

### Sync Manual Trigger

1. User clicks "Forçar Sincronização" inside Conectores card
2. Button: disabled + icon changes to `sync` + text "Sincronizando..."
3. API call to `/api/admin/sync-connector` (POST)
4. On success: toast.success "Sincronização concluída — N transações importadas" + connector status updates to "Conectado"
5. On stub error (expected in this phase): toast.info "Conector Pegasus aguarda credenciais" — not toast.error, since stub behavior is expected
6. On real error: toast.error "Falha na sincronização" + error detail

## Layout Notes

### relatorios/page.tsx grid

Current: `grid grid-cols-1 md:grid-cols-2 gap-6 max-w-4xl`

Phase 06 change: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6 max-w-6xl`

All three cards use identical glass-card structure. No new layout primitives needed.

**Focal point declaration:** All three cards (Export, Import, Conectores) are co-equal utility actions with equal visual weight — identical glass-card shell, same icon container size, same title size. No single dominant element exists on this page. Reading order (left-to-right, top-to-bottom) establishes implicit priority: Export > Import > Conectores. This is intentional — users arrive for different tasks at different times, so no card is elevated above the others.
