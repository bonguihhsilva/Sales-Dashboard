# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **shadcn note:** `components.json` does not exist yet — shadcn init is a deliverable of this phase (FOUND-03).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, tick marks |
| sm | 8px | Compact element spacing, label-to-value gaps |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Section padding, card-to-card gaps |
| xl | 32px | Layout gaps, sidebar internal padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Sidebar width collapsed: 64px (aligns with 3xl token — intentional)
- Sidebar width expanded: 240px (fixed layout constraint, not a spacing token)
- Mobile touch targets: minimum 44px height on all interactive elements (WCAG 2.5.5)
- Topbar height: 56px (14px multiple, standard topbar proportion)

> **Implementation note:** Scrollbar width is 6px — inherited from existing globals.css (aesthetic only, not a spacing token).

## Typography

Two fonts only. Body uses Syne; data/labels/numbers use DM Mono.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Syne | 28px | 700 | 1.2 | Page titles, hero values in stat-card |
| Heading | Syne | 20px | 700 | 1.25 | Section headings, page-header title |
| Body | Syne | 16px | 400 | 1.5 | Navigation labels, button text, descriptions |
| Label | DM Mono | 12px | 400 | 1.4 | Stat-card labels, table headers, badge text, breadcrumbs, metadata, table cell values, numeric data, period labels |

Rules:
- Never use Syne below 16px
- Never use DM Mono above 12px
- All monetary/numeric values: DM Mono 12px weight 400
- All section labels and uppercase metadata: DM Mono 12px, `letter-spacing: 0.08em`, `text-transform: uppercase`
- Data-table body rows: Label (DM Mono 12px) for all cell content including numeric values

**Note:** Old tailwind.config.js values (`--jebai: #7b61ff`, `--mkt: #42d9f5`, `--caixa: #f5a742`) must be replaced with the finalized tokens above during this phase.

### Accent Reserved For

The `--accent` color (`#c8f542` dark / `#16a34a` light) is reserved exclusively for:

1. Active navigation item indicator (sidebar active state — left border or background)
2. Primary CTA button background (`Button` variant="default")
3. Progress-ring fill when goal is met or exceeded
4. Focus ring on interactive elements (outline style)
5. Positive variation indicator in stat-card (upward trend arrow + number)

Accent must NOT be used for: headings, body text, decorative elements, loading states, or any element not in the list above.

## CSS Variable Migration

The existing `globals.css` uses legacy variable names. Phase 1 must rename all tokens:

| Old name | New shadcn-compatible name |
|----------|---------------------------|
| `--bg` | `--background` |
| `--text` | `--foreground` |
| `--surface` | `--card` (for card contexts) |
| `--surface2` | `--muted` |
| `--border` | `--border` (keep, update value) |
| `--muted` | `--muted-foreground` |
| `--accent` | `--accent` (keep, update value to #c8f542) |
| `--jebai` | `--jebai` (keep, update value to #9333ea) |
| `--mkt` / `--caixa` | delete — replaced by `--paje1` / `--paje2` |
| `--meta1/2/3` | keep as aliases → `--accent`, `--paje1`, `--paje2` |

All 100+ inline `style={{}}` references to old variable names must be replaced with Tailwind utility classes using the new tokens. New components must never use inline styles.

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Sidebar collapse | CSS transition 200ms ease on `width`, icons and labels fade |
| Sheet open | shadcn Sheet default animation (slide from left) |

## Implementation Notes for Executor

1. **Init order matters:** `npx shadcn@latest init` → override `globals.css` tokens → install components → build 7 custom components → build layout.
2. **No inline styles in new code:** All new components use Tailwind utility classes referencing the CSS vars declared above. The existing components in `src/components/ui/index.tsx` will be refactored to Tailwind as part of this phase.
3. **Font loading:** Replace `<link>` CDN tag in `layout.tsx` with `import { Syne, DM_Mono } from 'next/font/google'`. Apply via `className` on `<html>` or `<body>`.
4. **next-themes:** Install `next-themes` for theme toggle. Wrap layout in `ThemeProvider` with `attribute="class"` and `defaultTheme="dark"`.
5. **Sidebar state:** Collapsed/expanded state stored in `localStorage` via a custom hook — not URL state. Mobile sheet state is local React `useState`.
6. **Tailwind config update:** Replace old color tokens (`bg`, `surface`, `surface2`, `jebai: #7b61ff`, `mkt`, `caixa`, `meta1-3`) with shadcn-compatible CSS var references (`background: 'hsl(var(--background))'` pattern).
7. **Scrollbar width:** Keep `6px` scrollbar from existing `globals.css` as-is. This is an aesthetic detail, not a design system token.
